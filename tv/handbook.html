TV #1 — Menu Board

Menu AdminEdits the static menu board (the big TV behind the counter).

Open 192.168.0.133:8080/tv/admin.html in a browser. Sign in with your admin password. Click the TV #1 — Menu tab.

1. Common tasks

Change a price

  1. Find the item in the right category (e.g. Italian Grinders).
  2. Click the price field, type the new price (no dollar sign needed).
  3. Click Save Changes at the top right.
  4. The TV updates in about 30 seconds.

Add a new sandwich

  1. Scroll to the right category, click + Add Item.
  2. Fill in name, description, and price.
  3. Use the size and bold controls if you want it to stand out.
  4. Click Save Changes.

Take an item off the menu

  1. Click the × next to the item.
  2. Click Save Changes. (The item is gone — there's a backup on the Pi if you need it back.)

Make text bigger or bolder

Each item has small +/- size buttons and a bold toggle. Use these for daily specials or anything you want eyes on. Default size is 100; keep most items there.

Don't see your change? Wait 30 seconds and look again. The TV polls for updates on a timer, so changes don't show instantly — that's normal.

TV #2 — Rotating Screen

Rotation AdminEdits the rotating screen — slides, videos, deals, and the Pepsi clip.

Open 192.168.0.133:8080/tv/admin.html, sign in, and click the TV #2 — Rotation tab.

1. The rotation

This screen plays a list of slides one after another. Each slide is shown for about 10 seconds. You can reorder, add, or remove slides anytime.

  • Up / Down arrows reorder a slide in the rotation.
  • × removes a slide. (The file stays on the Pi for a day in case you put it back.)
  • + Menu Section adds a built-in slide (Soup Specials, Combo Deal, Featured Sandwich, etc.).
  • + Image uploads a photo (jpg / png / webp).
  • + Video uploads a clip (mp4 / mov). Used for the Pepsi spot.

2. The "Promos" section

Three slides on TV #2 read live data from the Promos panel — same admin page, scroll down on the TV #1 tab.

  • Featured Sandwich — pick an item from the menu, optionally upload a hero photo.
  • Combo Deal — name, items list (one per line), price, tagline.
  • QR / Reviews — paste your Google review URL; the slide generates the QR code automatically.

3. The Pepsi clip

Upload your Pepsi .mp4 with + Video, then drag it into the spot you want — most natural is right before Make-It-A-Meal.

Tip: TV #2 takes about 90 seconds to pick up changes — longer than TV #1 because it has to download new media.

Public Website

Website AdminEdits the public homepage at bankstgrinders.com.

Open 192.168.0.133:8080/tv/site-admin.html in a browser. Save writes the change locally on the Pi. Publish to Site sends it live to the world.

Important: the website doesn't change for customers until you click Publish to Site. Saving alone just stages the change — that's on purpose, so you can preview before going public.

1. Hours of operation

Each day has an open time, a close time, and a Closed checkbox. Use formats like 9:00 AM or 6:30 PM — half-hours work too.

2. Announcement banner

The big gold-bordered card at the top of the homepage. Empty = nothing shows. Type a message in plain language, hit Enter for line breaks. Pick an Auto-hide on date and the banner disappears after that day on its own — handy for "Closing early Friday" or "Open this Saturday."

Examples

  • "Closing early today at 3pm. See you tomorrow!"
  • "Open this Saturday 9am–2pm — come grab a grinder!"
  • "Closed Thanksgiving and Christmas. Back open December 26."

3. The Save / Publish flow

  1. Make your edits on the Site Admin page.
  2. Click Save — saves on the Pi only. The button at the top tells you how many changes are waiting.
  3. Refresh bankstgrinders.com (or localhost:8080 on the Pi) to preview.
  4. Happy with it? Click Publish to Site. About 30 seconds later, the live site updates.

If something goes wrong

TroubleshootingThe most common problems and how to fix them yourself.

1. "I forgot the password"

The password lives in the Pi's settings, set when we first plugged the Pis in. There's no recovery from inside the admin — call whoever set them up. Write the recovery contact on page 1.

2. A TV is blank or stuck on the desktop

  1. Make sure the HDMI cable is fully seated at both ends. Most "stuck" TVs are a loose cable.
  2. Power-cycle the Pi: unplug, wait 10 seconds, plug back in.
  3. Wait about a minute for it to boot.

3. "I edited something but the TV didn't update"

Wait. TV #1 takes about 30 seconds to refresh; TV #2 takes up to 90 seconds. If it's been longer, check that you clicked Save Changes. The Save button briefly says "Saved!" when it works.

4. "Publish to Site failed"

Most likely the Pi can't reach GitHub. Check the deli's internet. The local Save still worked — your changes are safe on the Pi. Try Publish again in a few minutes.

If the error mentions "Remote has new commits": someone made a change on GitHub directly. Whoever set up the Pi needs to pull/rebase it.

5. "The Site Admin says publishing isn't set up"

The GitHub access token wasn't installed on this Pi yet, or it expired. On the Pi: open a terminal and run bash tv/setup-github-auth.sh. Tokens last about a year — write the next renewal date below.

GitHub token expiry tracker

6. "I can't reach any admin page"

  1. Check that your iPad or laptop is on the deli's Wi-Fi (not cellular).
  2. Try Pi #2's IP if Pi #1 isn't responding: 192.168.0.119:8080/tv/admin.html serves the same admin.
  3. If both Pis are unreachable, the router may be down — restart the deli's router.

7. Finding a Pi's IP at the deli

If the IPs on page 1 stop working (the router rebooted, the deli changed Wi-Fi, etc.), here's how to find a Pi's new address:

  1. Easiest: on Pi #1, attach a keyboard, type hostname -I, hit Enter. The first number that comes up is the Pi's IP.
  2. From the router: log into the deli's router admin page (usually 192.168.0.1 or 192.168.1.1), look for a "connected devices" or "DHCP clients" list. The two Pis show up as raspberrypi or whatever names we set during setup.
  3. From your phone: if you can already reach Pi #1 in a browser, the URL bar shows its IP — write it on the cover page. Pi #2 is almost always one number away.

Once you have the new IPs, scratch out the old ones on page 1 and write the new ones in — or open the handbook on the Pi and reprint, and the URLs auto-update.

8. Backups

Every menu save is backed up automatically on the Pi at tv/data/backups/ (last 30 saves). The website's edit history lives in GitHub — every change is a version you can roll back.
